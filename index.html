BEVFERLE: The Bus Emissions Visualisation Redesign

A React and Redux lightweight implementation of my research project

Coming to the end of my time with the BEVis project, I realised that the design of the codebase was limiting my feature development. In order to rectify this, I planned a transition to React and Redux to reduce the complexity of the code, and enable refactoring and reuse of more of the code developed for the project.

BEVFERLE is the product of ~40 hours development and planning time. BEVFERLE uses React components on the front end, Redux to handle state, Python with Flask for a backend API that can provide data from MongoAtlas, and D3 to create visualisations.

BEVFERLE is an ongoing project of mine, and the codebase is available on my Gituhb (BEVFERLE Main). Another advantage to this refactor is the ability to reuse code, such as the pure D3 implementations of the stream graph.

The current version of this tool is hosted on Heroku (my favourite PaaS), with a MongoAtlas Database storing the data. Link to Heroku

Note: Heroku is slow to load when first connecting as it very kindly sleeps apps for me when there is no active connection, saving me $$$

BEVFERLE's main screan once loaded. Showing filters on the left side menu, and a stream graph of emissions related to bus engine class.

The graph underneath is referred to as a context by Mike Bostock, creator of D3. It is like an overview that is used to zoom, brush and pan the graph above. This comes from the information visualisation concept of Focus+Context.

Bus Emissions Visualisation

Visualising a model of Wellingtons Public Transport Emissions

Visualising the data related to the public bus transport network in Wellington has been subject of many projects of mine for the last two years. This project has been in development since November 2020, and became my Engineering Honours project.

The Bus Emissions Visualiser is a tool that allows users to gain insights about a model of the Greater Wellington bus public transport network. The model used for a data source was created by 2 software development interns and myself at Greater Wellington over the 2020 summer.

The tool creates 3 different visualisations, with a plethora of tools at the users disposal to drill into the dataset and gain insights from it. One of many goals with the project is to aid planners with replacing vehicles with electrics. These tools are intended to facilitate insights related to 3 major user tasks in the system:

  1. Quantify Emissions by different class, route and vehicle
  2. Understand the factors that affect emissions in the model
  3. Sensitivity test the model by introducing fleet changes

The current version of this tool is hosted on heroku, with a MongoAtlas Database storing the data. Link to Heroku

The heatmap that can be created with BEVis. Routes are the y-axis, dates along the x. A square represents the relative emissions of a bus route for one day.

Note: The public version only has access to one month worth of data for two visualisations.

Traffic Visualisation

Network Resilience

Traffic Visualisation has been an ongoing collaboration between VUW and Beca Group Ltd. There are many goals of the research, one of these is to better quantify the performance of a transport network.

Network Resilience was a two part research project that investigated the performance of rural and urban transport networks. I worked on mainly the urban tool, and contributed to the other project occassionally.

The urban tool creates a visualisation of isochrones to show the change in commuter travel time depending on time of day. An example of isochrones that can be created with this tool are below.

A project report in the form of a poster is available here. This poster was created to communicate the motivation and goals of the research to tertiary level students to generate interest in the projects.

Sports Data Visualisation

The ANZ Championship

As a short university project, I created some visualizations using the D3.js library to take a deeper look at some of the results of the games that were played in the ANZ Championship between 2008-2013.

I decided to focus on team performance for each game, for each year. Using the visualizations I designed, one can find interesting relationships between teams with similar patterns of play, performance in the tournament and more.

You can find a live demo version hosted on GitLab pages here.

Creating Better Interfaces

Designing a Learning Management System

The learning manangement systems that we use at university are rather unfriendly to the user. Several repeat buttons, confusing and hidden operations, and a lack of guidance for the user. We looked at how we could improve the experience of all users from lecturers to students. This project was a group project, undertaken with 5 other group members.

We used Jakob Nielsens Discount Usability Engineer concepts to guide the development of features neccessary for revision. This included development of personas, use cases and usability evaluation tests with heuristics.
